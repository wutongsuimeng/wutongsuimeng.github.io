单例模式

2023-04-08 11:03:58
约 487 字 预计阅读 1 分钟 次阅读

文章目录

定义：确保某一个类只有一个实例，而且自行实例化并向整个系统提供这个实例。 单例模式能保证一个类只有一个实例。实现方法是通过使用 private static 来保存实例变量，使用 private 修饰默认构造方法，避免通过 new 创建多个实例。

单线程实现：

public class Singleton {
    private Singleton() {}
    private static Singleton instance;
    public static Singleton getInstance() {
        if(instance==null){
            instance=new Singleton();
        }
        return instance;
    }
}

多线程实现（双重校验加锁）

public class Singleton {
    private Singleton() {}
    //使用 volatile 关键字实现实例变量的可见性，防止其他线程访问到部分构造的对象
    private static volatile Singleton instance;
    public static Singleton getInstance() {
        if(instance==null){
	        // 对实例化过程加锁，确保同时只有一个线程能实例化
            synchronized (Singleton.class){
	            // 再次判断实例是否已经创建，防止一个线程已经进入加锁区域但是还未初始化实例变量时，另一个线程判断null为true，导致两个线程获得了不一样的实例变量。
                if(instance==null){
                    instance=new Singleton();
                }
            }
        }
        return instance;
    }
}

另一种方式是直接创建实例变量：

public class Singleton {  
    private Singleton() {}  
    private static final Singleton instance = new Singleton();  
    public static Singleton getInstance() {  
        return instance;  
    }  
}

单例模式的优势：

  1. 避免了频繁创建、销毁对象的性能消耗
  2. 保证一个类只有一个实例，适合于只能全局共享的资源

缺点：

  1. 难于测试
  2. 违反了单一职责原则（一个类应该只有一个职责），但是单例模式拥有两个：保证一个类只有一个实例；提供一个全局访问点。
