离线编译maven项目

2021-05-23 15:08:58
Java
约 839 字 预计阅读 2 分钟

本文介绍如何在离线状况下编译maven项目。

假设项目分布为：

D->Code ->project ->pom.xml：项目位置

|->Maven ->conf ->settings.xml：maven配置文件的位置

|->repository：存放依赖的位置

命令行编译

先配置maven的配置文件：

<!--settings.xml-->

<?xml version="1.0" encoding="UTF-8"?>

<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0 http://maven.apache.org/xsd/settings-1.0.0.xsd">
 
  <localRepository>D:\repository</localRepository>
  
  <offline>true</offline>

</settings>

localRepository设置本地仓库位置，offline设置离线模式。

然后在命令行中打开项目的位置，即pom.xml文件所在的位置下，使用命令编译项目：

$ mvn clean package -s "D:/Maven/conf/settings.xml"

-s设置maven配置文件的位置。

修复进入grub rescue的问题

2021-03-21 17:37:56
linux
约 193 字 预计阅读 1 分钟

重装window后，突然就进去不了linux系统，在命令行窗口下提示

error :file '/boot/grub/x86_64-efi/normal.mod' not found

解决办法:

# 列出所有磁盘
> ls
> (hd0,gpt1) (hd0,gpt2)
# 一个一个查看磁盘里面的内容,找到grub所在位置,如果有提示/boot,/grub,即很有可能便是grub的位置
> ls (hd0,gpt2)/
> ./ ../ boot/ dev/ 
# 设置为启动分区
> set prefix=(hd0,gpt2)/boot/grub
> set root=hd0,gpt5
> insmod normal
> normal

之后便会重启系统

进入到系统之后,重新安装grub

# 查看/boot/efi的挂载点
> df
# 安装grub
> sudo grub-install /dev/sda1
